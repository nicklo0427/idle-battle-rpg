# V2-1 正式規劃稿
## 地下城推進（Dungeon Progression）

---

## 1. 文件目的

本文件用於定義 v2 的第一個正式系統：**地下城推進（Dungeon Progression）**。

V2-1 的目標不是擴大遊戲類型，而是在既有 v1 MVP 基礎上，補足更完整的：

- 中期可玩性
- 推進感
- 區域辨識度
- 裝備追求
- 基地／冒險／鑄造連動

本文件應作為：

- V2-1 的唯一規格來源
- 後續 Claude / Claude Code 工單的依據
- 工程拆分、資料結構調整、UI 規劃的正式基準

---

## 2. 系統定位

### 2.1 V2 在整體產品中的定位

v1（Phase 1–12）已完成 MVP，具備：

- Base / Adventure / Character 三大主頁
- 採集 / 鑄造 / 冒險
- 商人系統
- 自動結算
- 角色成長
- onboarding / toast / first-session 收尾

v2 的主題已確定為：

**中期可玩性擴張（Meta Progression + Content Depth）**

v2 開發順序已定為：

1. V2-1：地下城推進（Dungeon Progression）
2. 裝備與鑄造深度（Equipment / Crafting Expansion）
3. 基地升級（Base Progression）
4. 角色培養深度（Hero Build Depth）
5. 每日目標與成就（Daily Goals / Achievements）

### 2.2 V2-1 要解決的問題

v1 的地下城已可玩，但仍偏向最小版本，缺少：

- 區域式推進感
- 首通里程碑
- Boss 重刷價值
- 區域裝備線
- 區域素材與鑄造的明確關聯
- 清楚的中期成長目標

因此，V2-1 的核心任務是：

**把地下城從「任務入口」提升為「有層級、有首通價值、有裝備追求、有區域 identity 的正式 progression 系統」。**

---

## 3. 設計目標

V2-1 必須達成以下目標：

### 3.1 建立推進感

玩家需清楚感受到：

- 自己正在往下一層推進
- 每一層首通都有意義
- 每區 Boss 首通是大里程碑
- 未解鎖內容是可見且可期待的

### 3.2 建立區域 identity

三個區域必須在以下面向具備明顯辨識度：

- 命名
- 氛圍
- 掉落物
- Boss 特材
- 區域裝備線
- 區域代表武器

### 3.3 建立裝備追求

玩家要能理解：

- 不同層會解鎖不同裝備部位
- 每區都有自己的裝備線
- Boss 有區域代表武器
- Boss 可重刷以追求更好的武器版本

### 3.4 維持 MVP 邊界

V2-1 的目標是做深，不是做大。

本專案仍維持為：

- iOS 單機放置 RPG
- 不做後端
- 不做帳號
- 不做雲端同步
- 不做排行榜
- 不做公會 / 社交
- 不做 PVP
- 不做即時戰鬥

## 4. 範圍定義

### 4.1 本階段要做

V2-1 可包含：

- 地下城區域推進
- 每區 4 層結構
- 4 部位裝備系統
- 區域裝備線
- Boss 武器 farming
- Boss 特材
- 地下城資訊透明化 UI
- Adventure 頁正式化結構

### 4.2 本階段不做

V2-1 不包含：

- 職業系統完整化
- 複雜 affix 系統
- reroll / reforge 系統
- 套裝效果大系統
- 複雜基地升級樹
- 每日任務 / 成就
- 後端
- 帳號系統
- 雲端同步
- 排行榜
- 公會 / 社交
- PVP
- 即時戰鬥

## 5. 核心規格總覽

### 5.1 區域數量

V2-1 共有 3 個地下城區域：

1. 荒野邊境
2. 廢棄礦坑
3. 古代遺跡

### 5.2 每區層數

每區固定 4 層：

- 第 1 層：一般層
- 第 2 層：一般層
- 第 3 層：一般層
- 第 4 層：Boss 層

### 5.3 可重刷規則

- 每層皆可重刷
- 每層首通具推進意義
- Boss 首通是區域里程碑
- Boss 重刷主要用於武器 farming 與 Boss 特材取得

---

## 6. 裝備系統調整

### 6.1 裝備部位升級

v2-1 將裝備部位由 v1 的 3 部位升級為 4 部位：

1. 飾品
2. 防具
3. 副手
4. 武器

### 6.2 副手規範

- 副手先作為通用 slot
- 暫不做職業化主手 / 副手拆分
- 後續若有職業深化，再另行規劃

### 6.3 解鎖節奏

每區固定採以下部位解鎖節奏：

- 第 1 層：飾品
- 第 2 層：防具
- 第 3 層：副手
- 第 4 層 Boss：武器

目的：

- 讓每一層首通都有成長回饋
- 讓推進節奏與裝備解鎖節奏一致
- 讓玩家在每區都能逐步建立完整裝備線

---

## 7. 武器系統規範

### 7.1 區域代表武器

每個區域固定有一把代表武器，由該區 Boss 掉落。

### 7.2 武器 farming 規則

- Boss 可重刷
- 同名武器可有浮動數值版本
- 玩家可透過反覆挑戰 Boss，追求更好的版本

### 7.3 本階段限制

本階段武器系統僅包含：

- 區域代表武器
- 基礎浮動數值 farming

本階段不做：

- affix
- reroll
- reforge
- 複雜詞條組合

---

## 8. 素材系統規範

### 8.1 區域小素材

每區第 1～3 層，各有一種專屬小素材，用於製作對應裝備部位。

### 8.2 Boss 特材

每區 Boss 會掉落一種專屬 Boss 特材。

Boss 特材用途：

- 區域武器核心材料
- 區域裝備升階材料
- 少量基地升級需求（僅保留接口，不在本階段展開大型基地樹）

### 8.3 製作需求規範

製作裝備時，需同時消耗：

- 該層小素材
- 基地採集通用素材
- Boss 特材（武器 / 升階用途）

目的：

- 建立地下城掉落與基地採集的連動
- 強化鑄造的中期價值
- 形成基地／冒險／鑄造的完整循環

## 10. 推進規則

### 10.1 解鎖原則

- 未解鎖區域需可見
- UI 必須直接顯示解鎖條件
- 玩家需提前知道後續內容與目標

### 10.2 首通價值

每層首通至少承擔以下一種價值：

- 開啟下一層
- 解鎖新的裝備部位
- 解鎖新的掉落池
- 建立該區裝備追求方向

Boss 首通則作為該區大里程碑，負責：

- 完成該區主推進
- 開啟代表武器 farming
- 形成區域完成感

### 10.3 重刷價值

一般層重刷價值：

- 補小素材
- 補特定部位製作材料
- 提供穩定鑄造資源

Boss 重刷價值：

- 拿代表武器
- 追求更佳武器浮動數值
- 累積 Boss 特材

## 11. Adventure 頁 UI 規劃

### 11.1 區域列表結構

Adventure 頁採以下結構：

- 顯示區域卡片
- 區域卡片展開後，列出該區 4 層
- 點擊某層後，開啟詳細 sheet / panel

### 11.2 詳細頁顯示內容

每個地下城層級的詳細頁，至少需顯示：

- 區域描述
- 推薦戰力
- 目前戰力 vs 推薦戰力
- 掉落表
- 掉落機率 %
- 首通狀態
- 該區已解鎖裝備部位
- Boss 材料是否已見過
- 挑戰 / 出征按鈕

### 11.3 資訊透明原則

地下城資訊需保持高透明度，至少明示：

- 掉落物
- 機率 %
- 推薦戰力
- 首通狀態
- 未解鎖條件

此原則的目的，是降低不確定感，強化玩家的規劃感與目標感。

## 12. 與既有系統的連動

### 12.1 與角色系統

- Character 頁裝備欄位需擴充為 4 部位
- 加入副手 slot
- 地下城推進與角色戰力成長需更直接連動

### 12.2 與鑄造系統

- Recipe / Craft 結構需支援 4 部位裝備
- 各區裝備需形成完整材料鏈：
  - 區域小素材
  - 基地通用素材
  - Boss 特材

### 12.3 與基地系統

- 基地採集提供通用素材
- 地下城提供區域素材與 Boss 特材
- 兩者共同支撐裝備製作與角色成長

V2-1 必須維持原專案核心：

**基地管理 + 個人冒險**

## 14. 命名與文件風格規範

### 14.1 命名風格

- 使用繁體中文
- 正式、克制、可產品化
- 有世界感，但不過度浮誇
- 採「地點感 + 氛圍感」命名
- 素材名需有來源感
- 裝備名需與素材名互相呼應

### 14.2 文件風格

- 結構化
- 可直接存成 markdown
- 清楚定義範圍與非範圍
- 不混入舊版衝突內容
- 以唯一正本為目標
